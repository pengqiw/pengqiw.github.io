论文写作与文献查阅

发表于2023-04-25|更新于2025-03-06|学习笔记
|浏览量:

论文写作相关问题

  1. 解决的问题描述清楚

  2. 算法流程图补充：在流程图下面分别对每个步骤进行阐述

  3. 验证方法使用的数据集描述清楚，有几组，以及如何划分训练及测试集

  4. 应用分析过程及图谱

  5. 方法对比列表，最好对应前人文献中的方法

  6. 方法应用分析结果：需要准确率、召回率等指标，高低的原因。

简单概括如下：

  1. 文章结构不合理；

  2. 文章重点不清晰；

  3. 缺乏数学公式支撑；

  4. 没有领略学术八股文的内涵

3.23日与对标论文之间差异总结：

  • 不得如何准确命、简洁名图表和图片的的方法；

  • 在图表分析部分语言不够丰富，当图表、图片较多时语言会重复较多；

  • 算法介绍部分冗余较多；

  • 存在许多对同一事物不同的表达，如：大块煤检测、皮带机大块煤检测；皮带机、皮带运输机；

  • 用词很空泛，如很大程度上、很好的效果；

  • 表述不够专业

4.7日经验总结：

  • 文献部分选取近几年好的期刊，做到每个文献一总结，总结尽量用期刊的话；

  • 综述部分是一个总分结构；

  • 理论基础部分也标参考文献；

  • 方法提出部分第一段，分析出当前要解决问题的核心，同时自己方法对应解决的点，注意分析问题的本质；

  • 方法描述部分结构应该是方法、公式、公式中变量含义；

  • 描述？先把关键点分点写下来，然后处理各个点之间的耦合关系

文献检索

  • 优先查综述，从综述中招和自己方向相近的关键文献，再从关键文献入手搭建文献网络

  • 好的综述不是文献的罗列，是分析和整理，是有对未来恰当的展望和过去问题方法的总结；

  • 读综述时一定要做笔记；

各版本改动分析

第一版内容

  • 2.20号出了第一版报告，瀑布式的结构，完全没有遵照论文格式；

  • 有很多冗余的内容，在基础理论部分不必要的流程图和冗余的介绍；

第二版内容（3.10）

  • 初步具备了论文的引言、基础理论和方法介绍几大部分；

  • 删除了一些次要的内容，比如选取方法时的对比实验、筛选阈值经验值的选定；

  • 流程图更新成了训练和部署两条路线；

  • 但还是没有对比实验、结论、摘要等部分；

  • 仍旧存在冗余部分（插值部分数据和图像）

第三版内容（3.16）

  • 进行了对比实验，加上了结果分析部分；

  • 结果分析部分分析的是算法的各个部分对结果的影响、与对比算法在训练数据上的对比和本文算法在泛化数据上的泛化效果；

  • 增加结论部分；

  • 增加公式号

第四版内容（3.25，关键改动）

  • 增加了摘要、关键词、题目等部分，对论文格式全面排版；

  • 调整部分内容的位置（如大块煤的定义放在绪论中）

  • 实验与结果分析部分改各部分分为训练和验证（应用效果）

  • 开始增加一些公式化描述

  • 继续调整语言准确性

第五版内容（4.03）

  • 流程图增加抽帧等内容；

  • 增加公式化描述

  • 调整了方法引出的一些内容；

第六版内容（4.10）

  • 算法引出，不用公式、流程图一眼看明白；

  • 结果分析介绍清除实验流程和使用数据集，图表精炼

  • 参考文献选取高水平

总结

首先要明确一点：论文写作是有套路的，需要些什么、怎么写都是有固定木板的。论文写作大概有几点核心要求：完整、规范、精炼准确、流畅和学术，每一点都有不同对应，现总结如下：

完整性要求

完整性要求包含两块内容：一是该有的内容必须有，二是冗余的内容必须剔除

  • 题目、摘要、关键词、引言、理论基础、方法介绍、试验及结果对比、结论、参考文献；

  • 每一部分对应内容也都要完整，比如引言的三段都要有，具体见小论文写作指南；

  • 除大块的结构要完整外，公式号、图号、表头也要完整；

规范性要求

  • 是否有错别字，是否有同一概念的不同表述，是否有自己“捏造”出的概念；

  • 图号、公式号、表号、参考文献号是否连贯且正确

  • 公式部分的字体是否为斜体，在文中对应部分是否对应斜体；

  • 排版格式：字号、缩进、行距、英文字体，尤其是表格内的格式是否正确；

  • 引文格式；

  • 综述部分尽量对所述每一个文献都进行分析而不是列出来许多分析一次，综述部分是总分格式；

精炼准确性要求

  • 整体要求

  • 避免使用空泛词，如很大程度上、很好的效果；

  • 紧扣待解决的问题和解决方法，去除冗余表达，尤其是方法介绍和综述部分；

  • 引言的第三部分

  • 一定是针对针么问题提出什么解决方法，有什么过人之处

  • 方法理论基础不需要很多的图和细节的描述，但一定要把参数讲明白；

  • 方法引出部分

  • 描述：从本质上分析所要解决的问题有哪些根源，然后提出自己的方法，逐点解决这些问题。不需要很详细的介绍算法流程，主要是对问题的解决，一般不需要公式和符号。

  • 流程图：需要深入理解自己所做的东西，力求做到有一张图就可以理解整个文章的流程。后续小节划分围绕流程图各个块展开；

  • 方法介绍部分

  • 公式化描述：

  • 参数介绍：在该部分介绍主要参数的设定依据和参考范围，在实验部分介绍具体数值

  • 在本文项目中的意义而非算法本身的意义

  • 实验数据介绍部分

  • 来源、类型、训练和测试集划分、样本容量等数据；

  • 最好有图片和图表描述

  • 图表分析部分

  • 对比分不仅要在指标上说明本文方法优秀，还要从方法解决本质问题去分析；

  • 表格数据要精练；

流畅性要求

  • 整体应该是层层递进，逐渐突出问题本质然后由问题本质到具体方法再到实验结果的过程。不应该突兀出现一个表达，要仔细考虑每个小块都应该归在那个大模块；

  • 检查有无病句，说人话；

  • 公式放在对公式的描述前（符合人类阅读习惯）

学术性要求

  • 选取高水平的引文；

  • 对比方法要与课题接近且为近几年高水平文献；
