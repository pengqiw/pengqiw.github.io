一、Fault Description Based Attribute Transfer for Zero-Sample Industrial Fault

针对的任务：

  • 同一系统中存在部分故障缺失，通过学习其他已有故障及其属性的关系来建立属性向量到故障样本的映射。在推理时利将新的故障案例映射到属性向量，并根据属性向量的取值进行诊断推理。

  • 延伸为拥有同样故障和故障属性的同类型系统，对系统A的各故障样本进行学习，在系统B上进行推理；

当前问题：

  • 数据驱动的方法对各健康状态样本数量都有比较高的要求；

  • 生产过程中较难获取故障状态数据

现有研究成果：

  • 深度迁移学习，存在问题：对于深度迁移学习，在训练领域已经很好地准备了相同类型的故障，虽然目标域中没有对故障样本的需求

  • 故障树，存在问题：需要对系统中基础故障的发生频率有较为详细的记录

  • 零样本学习在图像领域有成功应用，分为两类：直接属性预测和间接属性预测，存在问题：需要更强有力、更适合的属性来应用于故障诊断领域；

对故障的描述：

  • 故障的影响；

  • 故障的位置；

  • 故障的原因；

属性的描述方法：

  • 共有C种不同的属性，因此每种故障的属性向量为一个C维向量，当有某种属性时该位置值为1，其他位置的值为0；

  • 使用时将属性向量与标签向量融合，使用监督学习方法从训练样本提取出与融合向量对应的特征**（条件限制特征提取）**

  • 特征与属性对应关系学习，通过监督学习方法学习对应关系，完成从特征向量到属性向量的映射；

  • 基于属性向量进行诊断推理

知识形式：故障征兆组成的one-hot属性向量

知识利用方式：特征映射到属性向量，基于属性向量进行分类

借鉴之处：

  • 建立特征和属性向量之间的映射关系；

  • 在实验数据不慎完美时尽量扬长避短，重点介绍较好的指标，对不好的指标进行补充解释甚至补充实验，

  • 在没有同类型方法时对其他领域方法进行适当修改或对本领域其他方法进行一些条件限制，核心要点是尽量控制单一变量；

不足和改进方向：

  • one-hot向量没有充分利用故障描述的语义信息，可以改用词向量嵌入；

  • 直接用于分类有些过于粗暴，可以用更好的方式利用知识；

二、Hybrid attribute conditional adversarial denoising autoencoder for zero-shot classification of mechanical intelligent fault diagnosis

与第一种方法应用场景相似，但是所用知识（故障描述）、知识的表示形式（SA和NSA结合）和知识的利用方法（与潜变量拼接）与第一种方法不同。其所提出VAE和故障描述利用的方法有较高的借鉴意义。

当前问题：

现有的数据驱动的诊断·方法包括无监督和半监督方法均需要充足的的数据；

现有研究成果：

  • 小样本诊断模型，存在问题：真实情况下，某台具体设备往往只有健康状态数据，甚至有些恶性事故不能被观测到，不满足小样本的要求；

  • 零样本学习在分类问题上取得一些成功的应用；

  • 作为零样本学习的中间件，SA（语义属性）相关的研究的发展也大力推动了零样本学习的进步和发展；

  • 在各类方法中，特征生成模型可以显著提高分类任务的效果，生成模型使用方法主要为VAE系列和GAN系列

所提方法：

  • 使用AAE的架构，在DAE中加入Discriminator，控制因变量的分布；

  • 在潜变量层拼接混合属性（语义和非语义属性）向量，共同作为生成器的输入

语义属性选择标准

  • 包含足够区分于其他类别的信息；

  • 可以从原始信息中观测到；

  • 能通过这些属性推断出该类别；

知识形式：

  • 语义属性：故障类型、故障程度、转频和负载等故障描述映射到[0,1]区间；

  • 非语义属性：CNN特征，实验结果表明非语义属性中包含信号时域和频域特征和其他信号典型特征

借鉴之处

  • 属性向量和隐变量拼接作为生成模型的输入；

  • 语义属性和非语义属性共同作为限制条件；

  • 其中语义语义属性为故障描述，非语义属性可以为信号的数字特征

  • 直接将健康状态样本作为VAE的输入使用属性向量作为限制条件时是否会比GAN有更好的效果；

不足之处：

  • 直接用CNN提取的特征作为非语义属性是否有充足的合理性？

  • 组件较多，推理过程较为复杂

三、Imbalanced sample fault diagnosis of rotating machinery using conditional variational auto-encoder generative adversarial network

可用于该设备其他工况已有部分数据的条件下进行增强

当前问题：

  • 风力发电设备齿轮箱工作在变载荷和变转速工况下，工况较多。每种工况都有大量正常状态监测数据，但是故障数据难以覆盖所有工况。

现有研究

  • 基于数据的方法，利用SMOTE等方法生成数据量较少的类别的样本

    • SMOTE在“非凸”故障样品空间和工作条件变化较大的情况下，可能会产生不良样品或无效样品。

    • 现有的分类网络降低了扩展数据集中的分类性能。

    • 当少数类样本与多数类样本之间的不平衡样本比例过大时，改进算法的效果会变小。

  • 基于算法的方法，成本敏感的集成学习方法

  • GAN模型生成故障样本，上述文献中涉及的样本生成方法可能会仅使用均方来评估生成样本和原始样本的误差来生成噪声样本且在训练过程中很难收敛，导致生成样本与真实样本差别很大

所提方法

  • 以cVAE为骨架，cVAE的Decoder作为cGAN的Generator。同时，在Discrininator中引入分类误差；

  • 与上一篇均使用了AE结构和GAN，区别在于上篇文章中GAN用于判别编码器得到的潜变量是否为正态分布，而本文是直接使用VAE，GAN的使用目的是判别VAE的生成样本与真实样本的差异，丰富损失函数增强训练能力。同时本文增判别器处加了分类误差和更广泛的c的使用（编码器输入、潜变量拼接输入生成器）

  • 当生成对抗学习到纳什均衡后可以利用生成器生成样本数较少的类别的样本，补充样本后再训练分类器以取得较好地分类效果；

知识运用方法

  • 所用先验知识为故障类别标签，该类别为包含了工况（转速、工作载荷）的类别标签；

  • 在编码器输入、生成器输入、判别器真实样本处均拼接该标签；

借鉴之处

  • cVAE+对抗学习+分类误差的模式很好，但是对先验知识的运用简单且粗暴；

  • 模型介绍部分由部分到整体，例如先模型结构再模型训练最后是整体诊断流程

  • 为排除深度学习随机性和训练样本选择引起的影响，可设计多组实验选择并绘制和分析误差棒；

  • 对生成样本的评价可以使用PCA等降维方法，绘制各个类别的聚类散点图；

  • 其他方法不方便直接比较时可以进行相应的改造（eg.对比其他生成模型时可以为其增加分类器）

  • 当所提方法有多种用途时（例如样本不均衡和部分工况样本缺失两种应用场景），可以分两个案例写；

四、Improved adversarial learning for fault feature generation of wind turbine gearbox

当前问题

  • 风机应用广泛，工作环境恶劣容易发生各类故障，且一旦故障发生会导致经济损失并威胁人员安全；

  • 现有基于机器学习的故障诊断方法取的较好成果，但都需要大量样本；

现有研究

  • 降采样（under-sampling）、过采样（oversampling）和Smote等方法进行数据增强。但这些方法存在在如下问题：

    • 在紧邻选择中存在一定的盲目性（这一点不理解）

    • 处理类别样本不均的样本时效果较差，容易出现分布边缘化问题

  • 相较于其他方法，GAN能生成更逼真、更清晰的样本，在故障诊断领域也有较多应用，但GAN仍有如下缺点

    • 对抗训练过程容易发生模式崩塌，训练较为困难；

    • 生成器选择困难

    • 生成样本缺乏多样性；

    • GAN是一种无监督学习方法，无法生成特定的类别

所提方法

  • 使用小波包变换提取出的特征作为生成的对象；

  • 使用c-GAN来生成不同类型的故障样本，其中GAN使用Wasserstrin距离；

  • 基于生成的故障状态样本训练随机森林算法进行作为诊断模型；

c-GAN的条件仅仅为故障类型，因此只能适用于样本增强

五、A case study of conditional deep convolutional generative adversarial networks in machine fault diagnosis

研究背景：

在生产过程中，为及时发现和处理故障部件往往会导致较为严重的后果，因此开展故障诊断研究势在必行。

现有研究

  • 传统故障诊断方法通过信号处理和特征提取（时域、频域、时频域）方法从原始监测信号中采集有效特征进行分析来实现，但是这一方法过度依赖专家知识；

  • 传统机器学习方法，也需要大量的先验知识指导特征提取，此外由于收集的信号可能由来自多个组件的振动组成，因此很难从复杂的机械中提取代表性特征

  • 随着计算机科学、数据科学的发展和各类传感器的大量部署，“大数据科学”开始兴起，为了进行准确高效的故障诊断，包括深度学习在内的大数据方法也被应用于故障诊断领域。但是这些方法都建立在各个类别的样本数量平且充足衡的基础上。在现实中，很难获得足够的故障样本，因为在实际制造环境中，正常状态数据比故障状态数据更普遍

  • 传统的解决样本不均问题如过采样、集成学习等采样方法无法充分挖掘数据信息且数据生成方式倾向于线性；

  • GAN生成能力强且属于无监督学习，不需要样本标注

方法介绍

  • 使用c-DCGAN作为样本生成模型，直接生成时域样本，样本长度为500；

  • 在c-DCGAN中加入BN层，避免不良的初始化效果，同时保持梯度传播到每一层，并防止生成器收敛到同一点

  • 在生成器的输出层中，Tanh函数作为激活函数;在其余层中，ReLU 函数用作激活函数。在判别器的输入层中，使用Leaky ReLU函数作为激活函数;Sigmoid 函数用作输出层中的激活函数。

  • 生成器和鉴别器采用不同的学习率

  • 为了防止过拟合，加入Dropout和Weight Decay

借鉴之处

  • 介绍方法时需要划分各个环节，界定好各环节的输入和输出，将输入输出的映射以合适的公式表示；

  • 改文章创新点较为薄弱，神经网络的理论基础部分写的较为详细，神经网络应用在故障诊断部分也较为详细（但公式使用较为混乱）有一定参考价值；

六、The Multiclass Fault Diagnosis of Wind Turbine Bearing Based on Multisource Signal Fusion and Deep Learning Generative Model

研究背景

风电领域需要更高效、准确的故障诊断方法

现有研究

  • 传统故障诊断的流程：信号采集、信号处理、特征提取、降维和分类，传统的故障诊断研究需要降噪、滤波、特征提取等复杂过程。该过程依赖于专业的背景知识，消耗了巨大的人力，提取的特征的好坏直接影响故障诊断的效果，在面对复杂条件下的轴承故障诊断时，手动设计特征的难度大大增加

  • 通过使用大量数据进行训练，深度学习诊断方法实现端到端的诊断智能诊断方法

  • 在实际运行中，一个部件存在多个可能的故障位置，在故障诊断中常采用多传感器监测多个位置的故障。当前大部分深度学习故障诊断方法基于一个测点的监测数据来进行训练，没有获有效利用多传感器采集到的故障信息；

方法介绍：

  • 转子旋转一周传感器采集411个点，选取784个点，进行归一化并变形为28*28，使用2D小波变换将融合信号转换为2D的特征图

  • 使用CVAE-GAN扩充故障状态的监测数据

  • 使用CNN作为诊断模型进行故障诊断

七、Generating HSR Bogie Vibration Signals via Pulse Voltage-Guided Conditional Diffusion Model（未完成阅读）

研究背景

  • 振动信号中包含了机械部件间复杂的相互作用，提供了观察内部机构和外部激励相互作用的整体视角。但是，利用复杂振动信号需要先进的分析技术，这些能这确保精确的数据对齐和不同数据类型的无缝集成。

  • 必须将不同的操作参数与特定的机械行为相关联存在较大难度，传统方法通过视频分析将振动与故障模式对应起来，它们的功效受到某些条件的限制，并且它们往往无法跟踪随着操作场景的变化而演变的故障特征

  • 最近研究的激增强调了数据驱动的振动信号分析方法，重点关注机器学习和深度学习范式，这些方法直接从测量中自动提取潜在特征，以评估健康状况。但是，这些方法在高铁系统中发现的实际可变条件下存在一定的泛化挑战。

研究基础

  • 通过学习和模拟训练数据的统计属性，可以生成与真实数据集相似的新数据，通过模拟各种正常和异常条件下的设备行为来提高诊断性能。但是，GAN训练过程容易发生模式崩溃和不稳定的训练。此外，由于GANs是隐式的，这些方法在整合多模态信息方面仍有困难，限制了它们在更复杂和各种操作条件下的适用性（如时变工况）

  • 扩散模型（DMs）已经成为一个强大而有前途的新型生成框架，它可以实现比GAN更好的生成质量和稳定性。

  • DMs在文字生产图像和文本图像特征表示匹配上的取得了很成功的应用，后续工作进一步说明使用条件提示引导可以实现高质量的图像生成并强调了扩散模型在描述下生成更准确的图像（如文本）方面的可控性和有效性的增强。

八、Auxiliary information guided industrial data augmentation for any-shot fault learning and diagnosis

研究背景：

工业过程分析中对多种故障进行诊断可以视作一种机器学习的分类问题，面临小样本和零样本等极端情况下如何构建训练模型的问题。

现有研究

  • 当前解决故障诊断任务中小样本和零样本问题的方法有两种：基于算法和基于数据。

  • 基于算法的方法通过能在少量样本下进行训练的分类器来实现，常见方法为半监督学习、迁移学习。

  • 基于数据的方法通过数据方法首先生成虚拟样本以增强数据集，然后使用具有丰富样本的增强数据集来训练传统的分类器。常用方法包括Over-sampling、和基于深度学习的生成模型。

  • GAN作为一种具有卓越的能力，可以为其他不同领域的数据生成精确和多样化的样本的模型，在各领域得到广泛应用。当这些方法只适用于解决小样本问题，无法处理零样本问题，同时在样本数极少（10个）的情况下也难以取得较好效果。

  • 当前方法无法在极少样本和零样本问题上取得很好表现的原因是：这些模型的构建和训练过程仅基于数据集本身，这限制了模型在训练集之外引入外部辅助信息。

方法介绍

  • we start from the perspective of synthesizing data, and mainly focus on introducing the extrinsic auxiliary information of all fault for guiding the generative networks, for the purpose of solving the any-shot problem in industrial fault diagnosis, especially the zero-shot problem, which is seldom concerned before in the industrial field（本文从综合数据的角度出发，主要侧重于引入所有故障的外在辅助信息，以指导生成网络，以解决工业故障诊断中的任意触发问题，特别是工业领域以前很少关注的零触发问题）

  • 该方法利用GAN作为生成模型，通过构造属性空间引入故障属性的外在辅助信息。通过构建故障属性到相应样本的映射关系，GAN的生成器可以合成所有故障的样本，包括训练集中不存在的故障。通过这些虚拟样本，增强数据集用于训练分类器，分类器可以访问原始序列集之外的故障诊断能力。这种使用故障属性处理任意触发问题的生成对抗模型称为 FAGAN。

  • 首先对数据集进行特征提取和降维处理，常用方法包括LDA、PCA和AE，然后进行数据归一化；接着在属性向量和辅助函数的指导下训练FAGAN，并基于训练好的生成模型生成可见和不可见类的样本用以训练诊断模型

  • FAGAN的判别器有两个输出层，一个做真伪二分类，另一个判断输入样本来自哪个类别的多分类

借鉴之处

  • 对方法的描述尤其是英语描述很准确恰当，可以作为参照，同时对零样本和广义零样本学习有一个比较好的背景介绍，也可以借鉴。
